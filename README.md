# everest-content

**The content SSOT for Everest Capital USA.** Markdown in, multi-format out.

Live site: **[content.zonewise.ai](https://content.zonewise.ai)**

## What this is

Every piece of Everest content — research intake, customer reports, investor deliverables, marketing — lives here as a markdown file with YAML frontmatter. Eleventy renders to HTML. GitHub Pages serves it. Future pipelines render the same markdown to PDF (Playwright), video (`everest-cinematic` via Veo 3), and social cuts.

**One source, many outputs.** Change the markdown once, every format updates.

## Architecture

```
everest-content/
├── design.md                    ← Brand tokens (Stitch schema). Single source of truth for visuals.
├── .eleventy.js                 ← Eleventy config. Parses design.md into template variables.
├── _layouts/
│   └── dono-v5.njk              ← Canonical wrapper. Reads design.md. Every page inherits this.
├── content/
│   ├── index.md                 ← Site root
│   ├── research/                ← Research intake (transcripts, repo evals)
│   │   ├── index.md             ← Hub with auto-generated list
│   │   ├── dylan-extraction-funnel.md
│   │   └── karpathy-skills-review.md
│   ├── reports/                 ← Customer deliverables (coming)
│   └── ...
├── assets/                      ← Images, downloads, etc. (passthrough)
├── pdfs/                        ← PDF exports (auto-generated, coming)
├── videos/                      ← Video exports (from everest-cinematic, coming)
└── .github/workflows/
    └── build.yml                ← Build + deploy to GitHub Pages
```

## Key principles

1. **Design tokens live in `design.md`, not CSS.** Change a color there, every page updates. Images generated by Nano Banana Pro, prompts sent to Veo 3, and UI generated by Stitch all read the same source.
2. **Content files never contain visual chrome.** Only the `dono-v5.njk` layout knows colors. Markdown files contain content, frontmatter, and semantic structure only.
3. **Every content type is just a frontmatter variation.** A research entry, a customer report, and an investor memo share the same layout and differ only in frontmatter.
4. **K3 surgical-changes applies.** Edits to this repo should touch only the content being changed. Drive-by layout tweaks are rejected.

## Adding content

1. Create a new `.md` file under the right subdirectory in `content/`
2. Add YAML frontmatter: `layout: dono-v5`, `title`, `date`, and any type-specific fields
3. Write the content
4. Push to `main`. Eleventy builds, Pages deploys, live in ~90 seconds.

## Local development

```bash
npm install
npm run serve        # live reload on http://localhost:8080
npm run build        # build to _site/
```
